# WinForms Migration Plan for 'Shooter Game'

## 1. Context
I have an existing C# OOP game called **Shooter Game** written originally in `.NET 6.0`. It uses a clean architecture with:
* **Core:** Engine, Controller
* **Models:** Enemies, Maps, Users, Weapons
* **Repositories:** Data storage/tracking for entities
* **IO:** Reader, Writer (currently Console-based)

Currently, it runs as a console application. 

## 2. Goal
I need to convert this into a **Windows Forms application** for a university OOP project. I want to preserve as much of my existing business logic and OOP structure as possible, while replacing the Console-based Input/Output and standard Game Loop with WinForms UI, Events, and Timers.

Please help me implement the following step-by-step plan.

---

## 3. Execution Phases

### Phase 1: Project Restructuring & Upgrade
1. **Upgrade Framework:** Update the `.csproj` to the latest .NET framework (e.g., `.NET 8.0-windows` or `.NET 10.0-windows`) and enable `<UseWindowsForms>true</UseWindowsForms>`.
2. **Library vs. UI Separation:** Keep the existing folders (`Models`, `Repositories`, `Utilities`) untouched. They will act as our backend class library.
3. **Refactor IO:** Ignore or remove the existing `ConsoleReader` and `ConsoleWriter`. We will replace these with UI interactions.

### Phase 2: Designing the UI Forms
Create the following forms:
1. **MainMenuForm:** A simple starting screen with "Start Game", "Select User/Weapon", and "Exit" buttons.
2. **SetupForm:** A form allowing the user to create a `User` (from `Models/Users/User.cs`) and select a starting `IWeapon` (Rifle, Shotgun, Sniper) before entering the map.
3. **GameForm:** The main gameplay screen. This will feature a large `Panel` or `PictureBox` acting as the `IMap`, where the action happens.

### Phase 3: The Game Loop & State Management (The Engine)
1. **Refactor the Engine:** The current `Engine.cs` likely uses an infinite `while(true)` loop. We need to refactor this to be driven by a WinForms `Timer` (`System.Windows.Forms.Timer`).
2. **Timer Tick:** On every tick (e.g., every 16ms for ~60 FPS), the `Controller` should update the coordinates of all entities (from `EnemiesCoordinatesRepository`), handle weapon logic, and check for collisions/health updates.

### Phase 4: Rendering & Drawing (The `GameForm`)
1. **GDI+ Drawing:** Use the `GameForm`'s `OnPaint` event (or a specific Panel's `Paint` event) to iterate through the `UsersRepository` and `EnemiesRepository`.
2. **Visuals:** Draw the player and enemies using `e.Graphics.FillRectangle` or `e.Graphics.DrawImage` at their respective coordinates.
3. **Projectiles:** Visually represent the bullets shot by the chosen weapons (Rifle, Shotgun, Sniper) based on their specific behavior.

### Phase 5: Handling Input
1. **Movement:** Handle `KeyDown` and `KeyUp` events on the `GameForm` to capture W, A, S, D for moving the `User` across the `Map`.
2. **Actions:** Map the Spacebar or Mouse Clicks to trigger the `Shoot()` method of the currently equipped weapon. Update the game state via the `Controller.cs`.
